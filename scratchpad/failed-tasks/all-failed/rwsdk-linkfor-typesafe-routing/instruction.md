# Type-Safe Routing with `linkFor` in RedwoodSDK

## Background
RedwoodSDK (`rwsdk`) is a server-first React framework for Cloudflare. Its router supports type-safe link generation through the `linkFor<App>()` helper exported from `rwsdk/router`. Given an `App` type (the default export of your worker), `linkFor` returns a `link` function that accepts a known route pattern and any required path parameters, producing a fully-typed `href` string.

A baseline RedwoodSDK starter app is already scaffolded at `/home/user/myproject`. The starter ships with:
- An entry point at `src/worker.tsx` exporting `defineApp([...])` with only `route("/", Home)` defined.
- A Home page at `src/app/pages/home.tsx` that renders a generic welcome screen and has no links to a profile route.
- A pre-existing helper at `src/app/shared/links.ts` that already calls `linkFor<App>()` and exports a `link` function — you may import and use that helper directly, or you may set up your own `linkFor` helper if you prefer.

There is no `/profile/:id` route yet, and the Home page does not yet use `linkFor` to render any anchor.

## Requirements
- Add a new dynamic route `/profile/:id` that renders a `Profile` page component.
- The `Profile` page must render the `id` URL parameter inside its HTML so that visiting `/profile/42` produces a page whose body contains the text `42` (and visiting `/profile/abc-123` produces a page whose body contains `abc-123`).
- Update the `Home` page (at `/`) so that it renders an anchor (`<a>`) whose `href` is generated by the `linkFor<App>()` helper using the route pattern `/profile/:id` with the parameter `id` set to `"42"`. The anchor must produce `href="/profile/42"` and have visible link text such as `Go to profile 42`.
- All routing must go through the existing `defineApp(...)` call in `src/worker.tsx`; do not introduce a separate routing system.

## Implementation Hints
- Use `defineApp`, `render`, and `route` from `rwsdk/worker` / `rwsdk/router` to wire up the new dynamic route.
- You can either reuse the existing `link` helper at `src/app/shared/links.ts` (which is already built on `linkFor<App>()`), or create a new one with the `linkFor` import and an `App` type derived from your `defineApp` default export.
- The route handler / page component receives `params` from RedwoodSDK; you can read `params.id` from it to render the URL parameter.
- Keep the existing project structure: put pages under `src/app/pages/` and keep the entry point at `src/worker.tsx`.

## Acceptance Criteria
- Project path: `/home/user/myproject`
- Start command: `npm run dev -- --host 127.0.0.1 --port 5173`
- Port: 5173
- Routes:
  - `GET /` — Returns HTML that contains an anchor with `href="/profile/42"` whose `href` value is generated via `linkFor<App>()` (`link("/profile/:id", { id: "42" })`).
  - `GET /profile/:id` — Returns HTML whose visible body contains the `id` URL parameter. In particular, `GET /profile/42` must return HTML whose body contains the text `42`, and `GET /profile/abc-123` must return HTML whose body contains `abc-123`.
- The `linkFor` helper must be imported from `rwsdk/router` and typed against the application's `defineApp` result, so that route patterns and required params are type-checked.

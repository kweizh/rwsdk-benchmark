Build a small comments API protected by a real CSRF (Cross-Site Request Forgery) defense using rwsdk's request pipeline (global middleware and/or per-route interrupters). Storage may be in-memory (per-process). * Provide a health endpoint that uses a safe method and is exempt from CSRF protection. ## Implementation Hints * See the RedwoodSDK docs at https://docs.rwsdk.com for `defineApp`, `route`, global middleware, and interrupters. Use a constant-time comparison if possible. * Cookies are read from the `Cookie` request header and set with `Set-Cookie` on the response. The framework's request object is a standard `Request`. * Keep storage trivial (a module-level array).

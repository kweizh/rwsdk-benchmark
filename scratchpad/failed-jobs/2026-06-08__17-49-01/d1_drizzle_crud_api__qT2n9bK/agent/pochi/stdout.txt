RedwoodSDK is a server-first React framework that runs as a Vite plugin on the Cloudflare Workers runtime. Persistence must be backed by a **Cloudflare D1** binding (running locally via miniflare in `wrangler dev` / the Vite Cloudflare plugin), accessed exclusively through **Drizzle ORM** (`drizzle-orm/d1`). A RedwoodSDK starter project is already scaffolded at the project path with `node_modules` pre-installed (including `rwsdk`, `drizzle-orm`, `drizzle-kit`, `wrangler`, and the Cloudflare Vite plugin). All routes must be grouped under the `/api` prefix using the rwsdk `prefix` helper (or equivalent route grouping). * All endpoints must read from and write to D1 through Drizzle. NEVER use in-memory state, files, or a fake DB. * Return JSON bodies with `Content-Type: application/json`. ## Implementation Hints * The rwsdk router is documented at https://docs.rwsdk.com/core/routing — use the object-form method handlers (`{ get, post, put, delete }`) for a single path. * Group routes with `prefix("/api", [...])` from `rwsdk/router`. * Drizzle + D1 integration is documented at https://docs.rwsdk.com/guides/database/drizzle — set up `src/db/schema.ts` and `src/db/db.ts` that exports a `drizzle(env.DB, { schema })` instance from `cloudflare:workers`. * The local D1 database for the dev server is materialized under `.wrangler/state/`. The database file must exist under `.wrangler/state/` after migrations are applied.

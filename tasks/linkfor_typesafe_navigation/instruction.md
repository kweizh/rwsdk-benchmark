# RedwoodSDK: Type-Safe Navigation with linkFor

## Background
A RedwoodSDK project is pre-installed at `/home/user/myapp`. Use `linkFor` from `rwsdk/router` to build a typed `link` helper, and use it inside server components so that every link in your HTML output is generated from the typed helper (no hard-coded path strings on the page).

## Requirements
Extend `src/worker.tsx` with the routes below and create a `src/app/shared/links.ts` file that exports a typed `link` constant created from `linkFor<App>()` (where `App` is the type of the default export from `src/worker.tsx`). Use that `link` helper inside the page components.

## Acceptance Criteria
- Project path: /home/user/myapp
- Start command: npm run dev -- --host 0.0.0.0 --port 5173
- Port: 5173
- Required file: `/home/user/myapp/src/app/shared/links.ts` must exist and export a value named `link`.
- Routes to implement (all under the existing `render(Document, ...)` block):
  - `GET /home`: server component HTML must contain an `<a>` element whose `href` is generated by calling `link("/about")` and whose visible text is `About`.
  - `GET /home`: server component HTML must also contain an `<a>` whose `href` is generated by calling `link("/users/:id", { id: "42" })` and whose visible text is `Show user 42`.
  - `GET /about`: HTML must contain an `<a>` whose `href` is generated by calling `link("/home")` and whose visible text is `Back to home`.
  - `GET /users/:id`: HTML must contain the text `User profile: <id>` (with the URL param substituted).
